## ТИПЫ КОММИТОВ
feat:     Новая функциональность
fix:      Исправление ошибки
docs:     Изменения в документации
style:    Форматирование, отсутствующие точки с запятой (без изменения кода)
refactor: Рефакторинг кода
test:     Добавление или исправление тестов
chore:    Изменения в процессе сборки или вспомогательные инструменты

## ОБЛАСТИ ПРИМЕНЕНИЯ
• frontend  - клиентская часть
• backend   - серверная часть
• database  - базы данных
• auth      - аутентификация
• ui        - пользовательский интерфейс
• api       - API endpoints
• config    - конфигурационные файлы

## ПРАВИЛА ОПИСАНИЯ
• Краткое описание должно быть не более 50 символов
• Используйте повелительное наклонение ("добавить", "исправить", "обновить")
• Не заканчивайте описание точкой
• Первая буква - строчная

## ПРИМЕРЫ ХОРОШИХ КОММИТОВ

fix(ui): исправить отображение кнопки на мобильных
docs(readme): обновить инструкцию установки
refactor(api): оптимизировать обработку запросов
test(backend): добавить тесты для модуля пользователей
